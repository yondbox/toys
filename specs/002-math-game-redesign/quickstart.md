**Date**: 2026-07-08 | **Plan**: [plan.md](./plan.md) | **Spec**: [spec.md](./spec.md)

実装後、この手順で受け入れシナリオ（spec の SC/FR）を端から検証する。詳細な契約は

## 前提・起動

```bash
pnpm install
pnpm dev            # http://localhost:3000/keisan-game
```

品質ゲート（マージ前に全て緑）:

```bash
pnpm lint
pnpm typecheck
pnpm test:unit
pnpm build
pnpm test:e2e       # 要 pnpm build
```

## 手動検証シナリオ

### US1 — 演算とレベルを選んで練習（P1）

1. `/keisan-game` を開く → ホームに 5 演算・3 レベル・れんしゅう／タイムアタックが並ぶ。
2. 「ひきざん」→「やさしい」→「れんしゅう」→ タイマー非表示・式・答え欄・テンキーが 1 画面に出る。
3. **テンキーだけ**（キーボードを使わず）で答えを入力→ OK。誤答は同じ問題のまま、正答で次へ
   （直前と別の問題）。→ SC-001/003、FR-005/007/012/013。
4. かけざん（九九）・わりざん（やさしい＝割り切れる）でも同様に成立。
5. わりざん「むずかしい」→ 「こたえ」「あまり」の 2 欄が出て、両方入れて判定。→ FR-006。
6. 「おわる」→ その回の正解数の成果表示 → ホームへ。0 問でも破綻しない。→ FR-016。

### US2 — タイムアタックと自己ベスト（P2）

1. たしざん・やさしい・10 もん → 3・2・1 の後に計測開始。進捗バーは正解のみ増える。→ FR-018/019。
2. 全問正解でタイム確定（正誤表示の時間を含まない）→ 結果に今回タイム。初回は「はじめての
   きろく」相当。→ FR-020/022。
3. 「もういちど」で再挑戦し、前回より速い → 「あたらしいきろく」演出＋記録更新。→ FR-023/024。
4. 別の演算・レベル・問題数で遊ぶ → 記録が混ざらない。ページ再読み込み後も記録が残る。
   → FR-021/025、SC-006。

### US3 — アプリ全体のダーク／ライト（P3）

1. OS/ブラウザをダークにして初回アクセス（`toys:theme` 未保存）→ 初回からダーク配色、
   誤配色のフラッシュなし。→ FR-026/030。
2. テーマトグルでライトへ → 表示中の画面が即変化。トップページ・`counter` など別ルートへ
   移動してもライト。→ FR-027/028。
3. 再読み込み・再訪 → OS 設定に関わらず選択（ライト）を保持。→ FR-029、SC-007。
4. 開発者ツールで `localStorage` を確認 → 書き込みは `toys:` 接頭辞のみ。既存の他おもちゃの
   キーが消えていない・上書きされていない。→ FR-031、SC-008。
5. プライベートブラウズ等で保存不能でも、プレイ・テーマ表示が既定で継続する。→ FR-032。

### モバイル最適化（SC-010/011）

1. 端末幅 375×667（たて向き）でプレイ画面を開く → 縦スクロールなしに収まり、テンキーが常に
   最下部に見える。→ FR-036、SC-010。
2. トップ・各おもちゃ・プレイの各画面で横スクロールが出ない（幅 375px）。→ FR-037、SC-011。
3. テンキー各キーのタップ領域が 48px 四方以上。→ FR-008。

## 自動テストの割り当て（目安）

- **Vitest（純粋）**: `operations.ts`（出題範囲・sub≥0・div 割り切れ/0除算禁止・あまり）、
  `game.ts`（遷移・正解のみ加算・計測確定・同問再挑戦）、`records.ts`（キー独立・破損
  フォールバック）。
- **Testing Library**: ホームの選択、テンキー入力／2 欄フォーカス、正誤表示、結果表示。
- **Playwright（`e2e/keisan-game.spec.ts`）**: US1〜US3 ジャーニー、375×667 で縦スクロール
  なし・横スクロールなし、`colorScheme` によるダーク初期化、手動切替のアプリ横断反映と再訪保持。
