# Feature Specification: たしざんタイムアタック

**Feature Branch**: `feature/speckit`

**Created**: 2026-07-06

**Updated**: 2026-07-07

**Status**: Draft

**Input**: User description: "小学生低学年向けに、フリーモードと10・30・50・100問の
タイムアタックモードで足し算を繰り返し解くゲームを作成する。"

**Revision (2026-07-07)**: バックスペースキーによる1文字削除、全画面状態での
レイアウト安定、2桁同士の足し算を出題する難易度「じょうきゅう」を追加。

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 自分のペースで足し算を練習する (Priority: P1)

小学生低学年の子どもはフリーモードを選び、画面に大きく表示された足し算の答えを
キーボードで入力する。打ち間違えたらバックスペースで1文字ずつ直すか、Escape で
全部消してやり直せる。正解するまで同じ問題に何度でも挑戦でき、正解後は自動的に
次の問題へ進んで、好きなだけ練習を続けられる。

**Why this priority**: 時間制限なしで「問題を見る、答える、結果を理解する、次へ進む」
という基本体験を成立させる、最小かつ独立したユーザー価値だからである。

**Independent Test**: フリーモードを開始し、入力のクリア、1文字削除、誤答後の再回答、
正解後の次問題への遷移を確認することで、このストーリーだけを独立して検証できる。

**Acceptance Scenarios**:

1. **Given** モード選択画面を表示している、**When** 子どもがフリーモードを選ぶ、
   **Then** タイマーを表示せず、足し算の式と空の答え欄を大きく表示する。
2. **Given** 答えを入力中である、**When** 子どもが Escape キーを押す、**Then**
   問題と進行状態を変えずに、入力中の答えだけを空にする。
3. **Given** 答えを2文字入力している、**When** 子どもがバックスペースキーを押す、
   **Then** 問題と進行状態を変えずに、入力中の答えの最後の1文字だけを消す。
4. **Given** 正しい答えを入力している、**When** 子どもが Enter キーを押す、**Then**
   大きな「〇」と正解を示す言葉を表示した後、新しい問題へ進む。
5. **Given** 間違った答えを入力している、**When** 子どもが Enter キーを押す、
   **Then** 大きな「×」と再挑戦を促す言葉を表示し、同じ問題を再び回答できる。
6. **Given** フリーモードで正解を続けている、**When** 子どもが次々に回答する、
   **Then** 問題数や誤答回数によって終了させず、練習を継続できる。

---

### User Story 2 - 決めた問題数の回答速度に挑戦する (Priority: P2)

子どもは10問、30問、50問、100問から挑戦する問題数を選ぶ。3秒間のカウントダウン後、
表示中のタイマーと進捗を見ながら足し算に回答し、選んだ数の問題をすべて正解すると
所要時間を確認して、同じ条件でもう一度挑戦できる。

**Why this priority**: フリーモードの基本体験に、開始制御、進捗、計時、完了という
独立した競技価値を加えるモードだからである。

**Independent Test**: 各問題数を選んだときに3秒のカウントダウン後から計時され、
選択した数だけ正解した時点で停止し、結果と再挑戦手段が表示されることを検証できる。

**Acceptance Scenarios**:

1. **Given** モード選択画面を表示している、**When** 子どもが10問、30問、50問、
   100問のいずれかを選ぶ、**Then** 「3、2、1」の順に1秒ごとのカウントダウンを
   表示し、カウントダウン終了後に最初の問題とタイマーを開始する。
2. **Given** タイムアタック中である、**When** 子どもが正解する、**Then** 正解表示後に
   進捗を1問増やし、完了数と選択した総問題数を示した状態で次の問題へ進む。
3. **Given** タイムアタック中である、**When** 子どもが間違える、**Then** タイマーを
   止めず、進捗を増やさず、同じ問題に再挑戦できる。
4. **Given** 選択した問題数の最後の問題に回答している、**When** 子どもが正解する、
   **Then** タイマーを停止し、選択した問題数と所要時間を結果として表示する。
5. **Given** タイムアタックの結果を表示している、**When** 子どもが再挑戦を選ぶ、
   **Then** 同じ問題数で新しいセッションを3秒のカウントダウンから開始する。
6. **Given** タイムアタック中に画面がカウントダウン、回答、正誤表示、結果と切り替わる、
   **When** 子どもがどの画面を見ても、**Then** モード選択へ戻る操作は同じ位置に
   同じ大きさで表示され続け、縮んだり欠けたりしない。

---

### User Story 3 - 上級者向けの2桁の足し算に挑戦する (Priority: P3)

計算に慣れた子どもは、モード選択画面で難易度「じょうきゅう」を選ぶ。じょうきゅうでは
2桁同士の足し算が出題され、フリーモードでも各タイムアタックでも、きほんと同じ操作の
まま最大3桁の答えを入力して挑戦できる。

**Why this priority**: きほんの全体験(出題、入力、判定、計時)をそのまま使い、
出題範囲と答えの桁数だけを広げる独立した追加価値だからである。

**Independent Test**: 難易度「じょうきゅう」を選んでフリーモードと任意のタイムアタックを
開始し、全問題が2桁同士で、3桁の答えを入力・判定できることを確認することで、この
ストーリーだけを独立して検証できる。

**Acceptance Scenarios**:

1. **Given** モード選択画面を表示している、**When** 子どもが難易度「じょうきゅう」を
   選んでフリーモードまたはタイムアタックを開始する、**Then** 以降の問題はすべて
   10から99までの整数2個の足し算になる。
2. **Given** じょうきゅうで回答中である、**When** 子どもが3桁の答えを入力する、
   **Then** 3桁すべてが表示され、4桁目の数字は受け付けない。
3. **Given** じょうきゅうのタイムアタック中である、**When** 選択した問題数まで正解する、
   **Then** きほんと同じように所要時間と完了数を結果として表示する。
4. **Given** 難易度「きほん」を選び直している、**When** 新しいセッションを開始する、
   **Then** 出題は0から9までの整数2個の足し算に戻る。
5. **Given** じょうきゅうのタイムアタックの結果を表示している、**When** 子どもが
   再挑戦を選ぶ、**Then** 同じ難易度・同じ問題数で新しいセッションを開始する。

### Edge Cases

- 答えが空の状態で Enter キーを押しても、正誤判定や進捗変更を行わない。
- 答えが空の状態でバックスペースキーを押しても、答えと進行状態を変更しない。
- 数字以外のキー入力(Enter、Escape、バックスペースを除く)は答えに追加せず、現在の
  問題や進行状態を変更しない。
- 答えの桁数が難易度ごとの上限(きほん2桁、じょうきゅう3桁)に達しているとき、
  追加の数字入力を受け付けない。
- カウントダウン中の数字、Enter、Escape、バックスペースの各キー入力はゲーム状態に
  影響させない。
- 正誤結果の表示中に Enter キーを連続して押しても、判定や進捗を重複させない。
- 正誤結果の表示中の数字、Escape、バックスペースの入力は、次の回答の答えに影響させない。
- 誤答しても問題数の進捗は増えず、同じ問題への回答回数を制限しない。
- 正解後に生成する次の問題は、直前の問題と同じ式を連続して表示しない。
- モード選択へ戻った場合は進行中のセッションを終了し、次回開始時に入力、進捗、
  タイマーを初期状態に戻す。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: システムは、難易度「きほん」「じょうきゅう」と、フリーモードおよび
  10問・30問・50問・100問の各タイムアタックを区別して選べるモード選択画面を
  提供しなければならない。
- **FR-002**: システムは、難易度「きほん」では0から9までの整数2個(答えは0から18まで)、
  難易度「じょうきゅう」では10から99までの整数2個(答えは20から198まで)を使う足し算を
  1問ずつ出題しなければならない。
- **FR-003**: システムは、現在の足し算の式と入力中の答えを、主要な操作や状態より
  明確に大きく表示しなければならない。
- **FR-004**: システムは、主キーボードとテンキーの数字入力を受け付け、入力した答えを
  即時に表示しなければならない。
- **FR-005**: システムは、回答可能な間は追加の選択操作を必要とせず、数字、Enter、
  Escape、バックスペースの各キーを受け付け続けなければならない。
- **FR-006**: システムは、Escape キーが押されたとき、現在の問題とセッション状態を
  維持したまま入力中の答えだけを消去しなければならない。
- **FR-007**: システムは、答えが入力された状態で Enter キーが押されたときにのみ、
  入力値を現在の問題の正解と比較しなければならない。
- **FR-008**: システムは、正解時に大きな「〇」と「せいかい」を0.5秒以上1.5秒以内
  表示し、その後に未完了のセッションでは新しい問題へ、完了したタイムアタックでは
  結果画面へ進まなければならない。
- **FR-009**: システムは、誤答時に大きな「×」と「もういちど」を0.5秒以上1.5秒以内
  表示し、答えを空にして同じ問題を再回答可能にしなければならない。
- **FR-010**: システムは、正誤表示中の重複判定と重複した進捗更新を防がなければならない。
- **FR-011**: フリーモードは、カウントダウン、タイマー、終了問題数を設けず、正解後も
  子どもがモード選択へ戻るまで次の問題を出し続けなければならない。
- **FR-012**: タイムアタックは、開始前に「3、2、1」を各1秒表示し、終了後にのみ
  回答を受け付けなければならない。
- **FR-013**: タイムアタックのタイマーは、カウントダウン終了時に0から開始し、
  最後の必要正解数に到達した時点で停止しなければならない。
- **FR-014**: システムは、タイムアタック中に経過時間を0.1秒単位で表示し、現在の
  正解数と選択した総問題数を常に確認可能にしなければならない。
- **FR-015**: タイムアタックでは、誤答中もタイマーを進め、正解した問題だけを
  完了数として数えなければならない。
- **FR-016**: システムは、選択した問題数に到達したら、所要時間と完了した問題数、
  同じ条件(難易度と問題数)での再挑戦、モード選択へ戻る操作を表示しなければならない。
- **FR-017**: システムは、プレイ中にモード選択へ戻れる明確な操作を提供し、戻ったときは
  進行中のセッションを終了しなければならない。
- **FR-018**: システムは、小学生低学年が理解できる短い言葉、見分けやすい高コントラストの
  色、十分な余白を用い、装飾より問題・答え・結果を優先して表示しなければならない。
- **FR-019**: システムは、正誤を色だけで伝えず、「〇」「×」とひらがなのメッセージを
  併用しなければならない。
- **FR-020**: システムは、直前に正解した問題と同じ2つの数を同じ順番で並べた式を、
  次の問題として連続表示してはならない。
- **FR-021**: システムは、バックスペースキーが押されたとき、現在の問題とセッション状態を
  維持したまま、入力中の答えの末尾1文字だけを削除しなければならない。答えが空の場合は
  何も変更しない。
- **FR-022**: システムは、入力できる答えの桁数を、難易度の答えの最大桁数(きほんは2桁、
  じょうきゅうは3桁)までに制限しなければならない。
- **FR-023**: システムは、モード選択、カウントダウン、回答中、正誤表示、結果表示の
  どの画面でも、また経過時間や進捗の表示更新の前後でも、モード選択へ戻る操作を含む
  各表示要素の大きさと位置を変えず、縮小・重なり・欠けを生じさせてはならない。
- **FR-024**: システムは、難易度をセッション開始時に確定してセッション中は変更させず、
  結果画面からの再挑戦では直前と同じ難易度を適用しなければならない。

### Key Entities

- **難易度**: 出題範囲と答えの桁数を決める設定。「きほん」は0から9までの整数2個
  (答えは最大2桁)、「じょうきゅう」は10から99までの整数2個(答えは最大3桁)を使う。
- **ゲームモード**: フリーまたはタイムアタックを表す。タイムアタックの場合は、
  10、30、50、100のいずれかの目標問題数を持つ。
- **足し算問題**: 2つの整数、画面に示す式、正しい答えを持つ1回分の出題。
- **プレイセッション**: 選択したモード、難易度、現在の問題、正解済み問題数、
  目標問題数、経過時間、カウントダウン・回答中・結果表示などの進行状態を持つ
  1回のプレイ。
- **回答試行**: 現在の問題に対して送信された答えと、その正誤結果。誤答はセッションを
  終了させず、同じ問題への次の回答試行を可能にする。

### Out of Scope

- 利用者アカウント、ログイン、複数端末間の同期
- 過去記録の保存、自己ベスト表示、オンラインランキング、他の利用者との通信対戦
- 引き算、掛け算、割り算、文章問題
- 「きほん」「じょうきゅう」以外の難易度、桁数の混在(1桁と2桁の組み合わせ等)、
  出題範囲の細かな変更
- 音声、効果音、キャラクター演出、報酬や収集要素
- 画面上の数字ボタンによる回答入力

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 初めて利用する対象年齢の子どもの90%以上が、大人の説明なしで15秒以内に
  フリーモードまたは任意のタイムアタックを開始できる。
- **SC-002**: モード開始後の回答入力、送信、入力クリア、1文字削除の各操作を、
  100%キーボードだけで完了できる。
- **SC-003**: 有効な回答を送信した全ケースで、正しい「〇」または「×」の結果が
  0.2秒以内に見分けられる状態で表示される。
- **SC-004**: どちらの難易度でも、10問、30問、50問、100問の各タイムアタックで、
  選択した数と実際に完了した正解数が100%一致し、最初の問題の回答開始から最後の
  正解までを計時できる。
- **SC-005**: 対象年齢の子どもの90%以上が、大人の補助なしで難易度「きほん」の
  10問タイムアタックを完了し、完了後に自分の所要時間を指し示せる。
- **SC-006**: 画面幅320から1440ピクセルの範囲で、拡大操作なしに問題、入力中の答え
  (じょうきゅうでは3桁)、正誤結果、タイムアタック中のタイマーと進捗が重ならずに
  判読できる。
- **SC-007**: プレイ開始から終了までの全画面遷移と経過時間表示の更新において、
  モード選択へ戻る操作の位置と大きさが変化するケースが0%である。

## Assumptions

- 主な利用者は、キーボード上の数字、Enter、Escape、バックスペースを扱える小学生の
  子どもである。難易度「きほん」は0から18まで、「じょうきゅう」は198までの数を
  読めることを想定する。
- 難易度はモードから独立して選び、フリーモードと各タイムアタックのどちらにも
  適用できる。難易度「じょうきゅう」は、2桁同士の足し算を学習済みか、挑戦したい
  子どもを対象とし、きほんと同じ操作体系のまま出題範囲だけを広げる。
- 1人が1台の端末で遊ぶことを想定し、対戦とは結果画面の所要時間をその場で他者や
  自分の別の挑戦と見比べることを指す。
- 誤答に時間加算などの追加ペナルティは設けず、再回答に要した時間そのものを
  タイムアタックの結果へ反映する。
- 問題は毎回新たに選ばれ、セッション間で同じ問題順を再現する必要はない。
- プレイ結果は現在の結果画面にだけ表示し、ページを離れた後まで保存しない。
