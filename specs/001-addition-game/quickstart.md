# Quickstart: たしざんタイムアタック 検証ガイド

**Date**: 2026-07-07 | **Plan**: [plan.md](./plan.md) | **Contract**: [contracts/ui-contract.md](./contracts/ui-contract.md)

実装後に機能が仕様どおり動くことを end-to-end で確かめる手順。実装コードは
[tasks.md](./tasks.md)(`/speckit-tasks` で生成)に従う。

## 前提

```bash
pnpm install
pnpm exec playwright install chromium   # 初回のみ
```

## 起動

```bash
pnpm dev
# http://localhost:3000 → トップ一覧に「たしざんタイムアタック」→ /addition-game
```

## 手動検証シナリオ

### US1: フリーモード(P1)

1. モード選択で「フリー」を開始 → タイマーなしで式と空の答え欄が大きく表示される。
2. 数字キーで入力 → 即時表示。3 文字目(きほん)は無視される。
3. `Backspace` → 末尾 1 文字だけ消える。空のときは何も起きない。
4. `Escape` → 全消去。問題は変わらない。
5. 正しい答え + `Enter` → 「〇 せいかい」が約 0.8 秒 → 別の問題へ自動遷移。
6. 誤答 + `Enter` → 「× もういちど」→ 同じ問題・空の答え欄に戻る。
7. 何問正解しても終了しない。「もどる」でモード選択へ。

### US2: タイムアタック(P2)

1. 「10もん」を開始 → 「3」「2」「1」が各 1 秒 → 最初の問題とタイマー(0.0)開始。
2. 正解ごとに進捗が `1/10 → 2/10 …` と増える。誤答では増えず、タイマーは止まらない。
3. カウントダウン中・正誤表示中のキー入力(数字・Enter・Escape・Backspace)は無効。
4. 10 問目の正解でタイマー停止 → 結果画面(完了数・所要時間・再挑戦・モード選択)。
5. 「もういちど」→ 同じ問題数でカウントダウンから再開。
6. **レイアウト**: カウントダウン→回答→正誤→結果のあいだ、「もどる」ボタンが
   縮んだり動いたりしない。

### US3: じょうきゅう難易度(P3)

1. モード選択で「じょうきゅう」を選びフリーモード開始 → 全問題が 2 桁同士。
2. 3 桁の答えが入力できる。4 桁目は無視される。
3. じょうきゅうの 10 問タイムアタックを完走 → 結果表示。「もういちど」で
   じょうきゅうのまま再開する。
4. 「きほん」に戻して開始 → 1 桁同士に戻る。

## 自動テスト

```bash
pnpm test:unit        # game.test.ts(規則・遷移)+ page.test.tsx(キーボード→表示)
pnpm build            # E2E の前提
pnpm test:e2e         # e2e/addition-game.spec.ts(US1〜US3 + SC-006/SC-007)
```

期待値の根拠は [contracts/ui-contract.md](./contracts/ui-contract.md)(表示要素・
キーボード・タイミング・レイアウト安定)と [data-model.md](./data-model.md)
(状態遷移表)を参照。

## 品質ゲート(マージ前に全て通す)

```bash
pnpm lint && pnpm typecheck && pnpm test:unit && pnpm build && pnpm test:e2e
```

## 確認ポイント(Success Criteria 対応)

- **SC-002**: 回答入力・送信・全消去・1 文字削除がキーボードだけで完了する。
- **SC-003**: Enter 後、〇× が体感即時(0.2 秒以内)に出る。
- **SC-004**: 選んだ問題数と結果画面の完了数が一致し、所要時間が表示される。
- **SC-006**: ブラウザ幅 320px と 1440px で要素が重ならない(DevTools で確認)。
- **SC-007**: どの画面遷移でも「もどる」ボタンの位置・大きさが不変。
