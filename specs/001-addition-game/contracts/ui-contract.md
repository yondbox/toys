# UI Contract: たしざんタイムアタック

**Date**: 2026-07-07 | **Plan**: [../plan.md](../plan.md) | **Data model**: [../data-model.md](../data-model.md)

このおもちゃが外部(利用者・E2E テスト・トップページ)に対して公開する観測可能な契約。
実装詳細(コンポーネント分割・クラス名)はここに含めない。

## ルート契約

- URL: `/addition-game`(Route Group `(toys)` は URL に現れない)
- トップページ一覧: `src/toys/registry.ts` のエントリ経由でリンクされる
  (title: たしざんタイムアタック)
- リロード・離脱でセッションは消える(永続化なし)

## 画面状態と必須表示要素

E2E・コンポーネントテストは `data-testid` でこれらを特定する。

| 画面 | 必須要素 (`data-testid`) | 内容の契約 |
|------|--------------------------|-----------|
| モード選択 `mode-select` | `difficulty-basic` / `difficulty-advanced` | 難易度の選択状態が見分けられる (FR-001) |
| | `mode-free` | フリーモード開始 |
| | `mode-10` / `mode-30` / `mode-50` / `mode-100` | 各タイムアタック開始 |
| カウントダウン `countdown` | `countdown-number` | 「3」→「2」→「1」を各 1 秒 (FR-012) |
| 回答中 `playing` | `equation` | `{a} + {b} =` 形式 (FR-003) |
| | `answer` | 入力中の答え。空でも領域を保持 |
| | `hud`(タイムアタックのみ) | `progress`(solved / target)と `timer`(0.1 秒刻み) (FR-014) |
| 正誤表示 `feedback` | `feedback` | 「〇」+「せいかい」/「×」+「もういちど」(FR-008/009/019)。オーバーレイ表示で下の要素を動かさない |
| 結果 `result` | `result-count` / `result-time` | 完了問題数と所要時間 (FR-016) |
| | `retry` | 同じ難易度・問題数で再挑戦 (FR-024) |
| | `back-to-modes` | モード選択へ戻る |
| 全プレイ状態共通 | `back-button` | モード選択へ戻る操作 (FR-017)。**全状態で同じ位置・同じ寸法**(モード選択画面では不可視だが領域は保持) (FR-023) |

## キーボード契約(playing 状態でのみ有効)

| キー | 条件 | 効果 | 由来 |
|------|------|------|------|
| `0`〜`9`(主キーボード / テンキー) | 桁数上限未満 | 答えの末尾に追加、即時表示 | FR-004, FR-022 |
| `0`〜`9` | 桁数上限到達 | 何も起きない | FR-022 |
| `Backspace` | 答えが 1 文字以上 | 末尾 1 文字を削除 | FR-021 |
| `Backspace` | 答えが空 | 何も起きない | FR-021 |
| `Escape` | — | 答えを全消去(問題・進捗・タイマー不変) | FR-006 |
| `Enter` | 答えが 1 文字以上 | 正誤判定 → feedback へ | FR-007 |
| `Enter` | 答えが空 | 何も起きない | FR-007 |
| 修飾キー併用(`Cmd`/`Ctrl`/`Alt` +) | — | ゲームは処理しない(ブラウザ既定動作を妨げない) | — |
| 上記以外のキー | — | 何も起きない | エッジケース |

countdown / feedback / mode-select / result 状態では、上記キーはゲーム状態を変更しない
(FR-010, FR-012)。モード選択・再挑戦・戻るはボタンのクリック/タップで操作する
(キーボード契約の対象は回答操作のみ = SC-002 の範囲)。

## タイミング契約

| 事象 | 時間 | 由来 |
|------|------|------|
| カウントダウン 1 ステップ | 1000ms(3 → 2 → 1) | FR-012 |
| タイマー開始 | カウントダウン終了の瞬間 = 0.0 | FR-013 |
| タイマー表示更新 | 100ms 間隔(0.1 秒単位表示) | FR-014 |
| 正誤フィードバック表示 | 800ms(定数 `FEEDBACK_MS`。0.5〜1.5 秒制約内) | FR-008/009 |
| 所要時間の確定 | 最終問題の Enter 判定時刻(フィードバック表示時間を含まない) | FR-013, SC-004 |

## レイアウト安定契約 (FR-023 / SC-007)

- 状態遷移(countdown → playing → feedback → result)および タイマー表示の更新の
  前後で、`back-button` と `hud` の bounding box(位置・幅・高さ)は変化しない。
- 正誤フィードバックはオーバーレイで表示し、`equation` / `answer` の位置を動かさない。
- viewport 幅 320px〜1440px で全必須要素が重なり・見切れなく判読できる (SC-006)。
